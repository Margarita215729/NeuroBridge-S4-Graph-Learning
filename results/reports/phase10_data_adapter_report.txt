Transformations applied
-----------------------

- schema validation and format detection (wide / long)
- column-name reconciliation to the standard longitudinal schema
- standardization into a long variable table
- self-baseline transformation (delta and percent change from personal baseline)
- unit-standardization placeholder: units are tracked and unsupported conversions are reported, not silently transformed (broad biomedical unit conversion is not yet implemented)
- variable-to-domain mapping
- per-domain aggregation into graph-ready domain scores (mean absolute self-baseline delta)

Graph-ready output
------------------

- adapter_generated_longitudinal_domain_scores.csv (wide, Phase-6 compatible)
- adapter_domain_scores_long.csv (long domain scores)
- adapter_domain_scores_wide.csv (wide domain scores)

Limitations
-----------

- variable-to-domain mapping is approximate and extensible
- template/example data are not scientific evidence
- unit conversion is limited unless explicitly implemented
- domain scores depend on which variables are available
- missing data can reduce domain coverage
- no clinical interpretation is performed

Guardrails
----------

This adapter validates and transforms HRP-like longitudinal data into graph-ready domain scores. It does not diagnose, score risk, infer exposure, or recommend treatment.
